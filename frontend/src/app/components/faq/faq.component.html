Frequently Asked Questions


What does signing in do?

Employees at Columbia University Irving Medical Center can log in using their MC account to get access to data for all (public and non-public) samples. Approved IGM affiliates can view the anonymized IGM IDs for samples and, if they are within the secured network, access additional information about the samples.


What are “Publicly Available Samples”?

These are samples whose consent agreement allowed them to be used as controls for other studies at the IGM. Signed-in users can turn off this option to view data from all samples. Anonymous/public users are restricted to data from these samples.


What does the “High Quality Variant” filter do?

This restricts results to variants that pass the following filters: coverage >= 10x, genotype quality (GQ) score >= 20, strand odds ratio (SOR) score for SNVs <= 3, strand odds ratio (SOR) score for Indels <= 10, fisher strand (FS) score for SNVs of <= 60, fisher strand (FS) score for Indels <= 200, RMS mapping quality (MQ) score >= 40, quality by depth (QD) score >= 2, quality score (QUAL) of >= 50, read position rank sum (RPRS) score <= -3, mapping quality rank sum (MQRS) score of >= -10, and GATK Variant Quality Score Recalibration filter (Filter) “PASS”, “VQSRTrancheSNP90.00to99.00” (likely), or “VQSRTrancheSNP99.00to99.90” (intermediate). Searches for genes or genomic regions have these filters applied by default.


What does the “Ultra Rare Variant” filter do?

This restricts results to variants that are absent in all queried external databases. Specifically, the variants’ allele frequencies (AFs) must be 0, 1 (if the reference allele is actually the minor allele), or NA. ATAVDB currently queries the following databases: ExAC, gnomAD Exome, gnomAD Genome, Genome Asia, GME Variome, Iranome, and TOPMED.


Which version of the human reference genome is ATAVDB data based on?

ATAVDB currently supports hg19. Future updates will include a version for GRCh38.
